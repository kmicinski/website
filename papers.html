Android

The "Redexer" binary rewriter: Android apps use very coarse permissions that allow them to access a very large set of system resources without the user's knowledge. Our thesis is that this is because is that the permissions are resource centric versus semantic centric. (Example: an app requests access to the internet rather than "cnn.com.") We show that a solution to this problem is to use Android's process separation to perform potentially risky operations in a separate process, and interface to this trusted deputy using RPC. We also present a binary rewriter which can modify apps "off the shelf" to retrofit existing Android apps to use our enhanced security policies.

  • Tech report: Dr. Android and Mr. Hide: Fine-grained security policies on unmodified Android. Jinseong Jeon, Kristopher Micinski, Jeffrey A. Vaughan, Nikhilesh Reddy, Yixin Zhu, Jeffrey S. Foster, Todd Millstein. CS-TR-5006, December 2011 [pdf]

The "Symdroid" symbolic executor: Symdroid is a symbolic executor that works on native Dalvik code (without going to JVM code). Our draft for the Science of Computer Programming outlines our approach to the construction of the symbolic executor.

  • Draft: SymDroid: Symbolic Execution for Dalvik Bytecode. Jinseong Jeon, Kristopher K. Micinski, Jeffrey S. Foster. [pdf]
